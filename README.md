Documentation 
 
GUARDIA is a finance personal budget tracker that helps users navigate the current financial landscape by enduring against the many expenses that befall most people in the modern era. The app design and inspiration come from Spartans. Building off Spartan colours, and the motifs of strength in the face of adversity, the app is designed to assist its users in guarding against the pitfalls that personal budgeting brings, by guarding their ambitions, and by providing a second look into their affairs.  
Guardia will help users make tracking their money easier, with the innovative features we added you can view your spending via a graph which will make it easier to analyze and view your finances, rather than a bunch of numbers which will be overwhelming to the user. 
To make it fun and exciting, we added a feature where you get an award when you stuck to your weekly goal that you set the week before. Features like these push users to be motivated to meet their weekly goals and get their award at the end of the week. 
We were tasked with incorporating features required by the POE which are as follows Database Usage, User Registration, Expense Categorisation, Photo attachments, Category reviewing and Daily spending. We also needed to add our own innovative features to the app which are as follows: 
•	Auto categorisation of expenses.
•	Automatic weekly goal assignments.
•	Category ranking with spending habits tracker.
•	Awards view for completing goals.

GitHub utilization 
While GitHub Actions organizes the "compilation" process, which is commonly referred to as transforming source code into a usable form, GitHub acts as the central repository for a personal budget tracker, storing code, tracking issues, documenting usage, and enabling version control.
By utilizing Actions, you can perform unit tests and linting with each push to identify mistakes promptly, create independent executables or bundle the tracker during release tagging, and produce regular reports (such as a daily or monthly spending overview) by retrieving data, running the analysis script, and saving the results.
Actions in web-based tracking systems can gather frontend resources (TypeScript, CSS bundling) and publish the active dashboard to GitHub Pages whenever there is a new commit to the main branch, offering complimentary and automated hosting.
Moreover, planned scheduled job triggers can generate up-to-date financial reports automatically, and GitHub Secrets safely keep banking integration API keys. Testing across different platforms, storing artifacts, and sending notifications through email or Slack also guarantee dependability, turning a basic script into a strong, automated tool for personal finance.
